**Language**
> JavaScript (Node.js)

Es el estándar para bots de Discord. Tiene la librería discord.js más madura, infinidad de ejemplos y es fácil de aprender. Corre en cualquier hosting gratuito.

</td><td align="left" width="50%">

**Framework / Librería principal**
> discord.js v14

La más completa con soporte para Slash Commands, intents y eventos. Elegimos la v14 por ser la actual y tener mejor rendimiento.

</td></tr><tr valign="top"><td align="left">

**Voz**
> @discordjs/voice

Librería oficial de Discord para manejo de voz. Reemplaza a las viejas (opusscript, node-opus) y funciona perfectamente con ytdl-core.

</td><td align="left">

**Streaming de audio**
> ytdl-core

Descarga solo el audio de YouTube sin necesidad de FFmpeg instalado (aunque en Replit ya viene). Liviano y confiable.

</td></tr><tr valign="top"><td align="left">

**Servidor web interno**
> Express

Necesario para mantener el bot vivo en Replit: al recibir pings de UptimeRobot, Replit cree que hay actividad y no duerme el proyecto.

</td><td align="left">

**Variables de entorno**
> dotenv

Para guardar el token de Discord sin exponerlo en el código. Es una mala práctica poner tokens a la vista.

</td></tr><tr valign="top"><td align="left">

**Hosting gratuito**
> Replit

Permite ejecutar Node.js 24/7 si recibe tráfico constante. Tiene consola integrada, fácil despliegue y no requiere tarjeta de crédito.

</td><td align="left">

**Monitor de actividad**
> UptimeRobot

Hace ping cada 5 minutos a la URL del Replit para que nunca duerma el proceso. Es gratuito y no necesita instalar nada.
